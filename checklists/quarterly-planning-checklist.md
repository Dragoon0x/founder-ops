# Quarterly Planning Checklist

Every 90 days. Don't skip the process; it's the forcing function.

## T-21 Days (Retro Prep)

- [ ] Quarterly retro scheduled (2-hour block, whole leadership)
- [ ] Metrics dashboard for the quarter prepared
- [ ] Goal vs. actual tracking finalized
- [ ] Customer feedback synthesized (calls, surveys)
- [ ] Team survey sent (engagement, blockers)

## T-14 Days (Retro)

### Retro meeting (2-3 hours)

- [ ] What we said we'd do (last quarter's OKRs)
- [ ] What actually happened (hits, misses, surprises)
- [ ] Honest assessment of why misses happened
  - Was the goal wrong?
  - Was execution wrong?
  - Was the environment different?
- [ ] What we'd keep doing
- [ ] What we'd stop doing
- [ ] What we'd start doing
- [ ] 3 biggest lessons

- [ ] Each person shares biggest personal win + biggest frustration

## T-10 Days (Strategic Framing)

- [ ] CEO + leadership sync on the ONE theme for next quarter
- [ ] North star metric confirmed
- [ ] Resource constraints identified (headcount, cash, time)
- [ ] External factors scoped (market, competitive, regulatory)

## T-7 Days (Draft OKRs)

- [ ] 3 Objectives drafted (qualitative, ambitious)
- [ ] 3 Key Results per Objective (measurable, specific)
- [ ] Owner for each KR (ONE person, not a team)
- [ ] Dependencies mapped

### Draft quality checks
- [ ] OKRs are ambitious (30-40% miss expected on well-set KRs)
- [ ] Not too many (3 Os, 3 KRs each = max; less is better)
- [ ] KRs are outcomes, not activities
- [ ] No vanity metrics
- [ ] Interdependencies acknowledged

## T-3 Days (Team Review)

- [ ] Share draft with team for input
- [ ] Specific concerns addressed
- [ ] Not "rubber stamp" — real revision based on feedback
- [ ] Dependencies with other teams explicit

## T-0 (Planning Day)

### Morning: Company OKRs finalized
- [ ] Present + discuss + lock
- [ ] What we're NOT doing this quarter (explicit kill list)

### Afternoon: Team / individual OKRs
- [ ] Each team translates company OKRs into theirs
- [ ] Each person maps top 2-3 individual goals
- [ ] Alignment check (do team OKRs sum to company OKRs?)

## T+1 Day

- [ ] OKRs documented in one shared place
- [ ] Tracking cadence set (weekly check-in, monthly score)
- [ ] Kickoff all-hands scheduled
- [ ] Communication to board about direction

## Throughout the Quarter

- [ ] Week 1: Kickoff, OKRs visible to everyone
- [ ] Week 2-4: Weekly progress updates
- [ ] Week 5-6: Mid-quarter checkpoint — adjust if needed
- [ ] Week 7-9: Push phase
- [ ] Week 10-11: Close-out, metrics wrap
- [ ] Week 12: Retro prep for next quarter

## Anti-Patterns

- OKRs set in a vacuum (no team input)
- Too many OKRs (10+ objectives = no priorities)
- Sandbagging (hitting 100% = goals were too easy)
- Setting and forgetting (never reviewing mid-quarter)
- Activity-based KRs ("ship feature X") instead of outcome-based ("feature X drives 20% activation")
