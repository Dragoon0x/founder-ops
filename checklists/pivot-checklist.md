# Pivot Checklist (Operational)

Use after the strategic decision to pivot is made. This is the operational 
side: what to do Monday morning through month 3.

## Week 0 (Decision Made)

### Day 0-1: Internal alignment
- [ ] Board informed (before team)
- [ ] Written summary of the pivot decision prepared
- [ ] Q&A anticipated for team
- [ ] Runway impact modeled (see `prompts/runway-scenarios.md`)

### Day 1-2: Co-founder / exec alignment
- [ ] All leadership on same page
- [ ] Each leader has their narrative
- [ ] Each leader's role in the new plan clarified

### Day 3: Team all-hands
- [ ] Written announcement in team doc (read before meeting)
- [ ] Live meeting (in-person or video; no Slack-only)
- [ ] Space for questions (honest ones, not rah-rah)
- [ ] 1:1 follow-ups with anyone who seemed rattled

### End of week 0: Broader network
- [ ] Key customers (if material impact)
- [ ] All investors (written email)
- [ ] Advisors / board members (individual)

## Week 1-2 (Wind Down Old)

### Product
- [ ] Feature freeze on old product
- [ ] Critical bugs only (no new features)
- [ ] Sunset timeline communicated to users

### Customers
- [ ] Old product users: transition plan
  - Option 1: Keep paying, we maintain for 12 months
  - Option 2: Refund pro-rata, graceful exit
  - Option 3: Open-source the old product
- [ ] Customer-by-customer plan for the top 10-20 accounts

### Team
- [ ] Team members who don't fit new direction: honest conversation
- [ ] Severance if needed (see `playbooks/firing-someone.md`)
- [ ] Remaining team: role clarification for new direction

### Ops
- [ ] Vendor contracts reviewed (cancel what's not needed for new direction)
- [ ] Marketing spend paused
- [ ] SEO content: keep or archive?
- [ ] Social accounts: brief explanation post

## Week 3-4 (Start New)

### Discovery (first)
- [ ] Customer interviews in new direction (20+ in first 2 weeks)
- [ ] ICP hypothesis on paper
- [ ] Problem validation (do these people actually have this problem?)

### Minimal product
- [ ] Prototype or MVP scope defined
- [ ] Week-by-week ship plan
- [ ] First 10 users identified (warm network)

### Messaging
- [ ] New 1-line: "What do we do now?"
- [ ] New landing page
- [ ] New positioning (see `modes/positioning.md`)

### Infrastructure
- [ ] Repo / codebase for new direction (fresh start or branch?)
- [ ] Tools / analytics tracking reset for new metrics

## Month 2 (Find PMF Signals)

### Build
- [ ] MVP in customers' hands
- [ ] Weekly ship cadence
- [ ] Customer feedback loop tight (direct Slack / calls)

### Measure
- [ ] Core metric defined (activation, retention, something that matters)
- [ ] Baseline numbers by end of month 2

### Narrative
- [ ] Board update with progress vs. pivot hypothesis
- [ ] Investor update: pivot results so far, what we're learning

## Month 3 (Inflection Point)

### The honest check
- [ ] Are users pulling harder than they did for old product at equivalent stage?
- [ ] Do customer interviews show unprompted pain?
- [ ] Is team more energized than they were 3 months ago?
- [ ] Are you starting to see repeatable acquisition?

### If YES signals
- [ ] Double down
- [ ] Hiring plan for next 90 days
- [ ] Public narrative (case studies, testimonials)

### If NO signals
- [ ] What's different from the hypothesis?
- [ ] Is this an execution issue or a new-pivot issue?
- [ ] Runway check (can we afford another pivot?)
- [ ] Honest conversation with board

## Month 4-6 (Execution)

### If pivot is working
- [ ] Standard company-building playbook resumes
- [ ] Series A story for new direction starting to form
- [ ] Old pivot narrative becomes the origin story

### If pivot is NOT working
- [ ] Second pivot consideration (usually a red flag — see `playbooks/pivot-decision.md`)
- [ ] Shutdown consideration
- [ ] Acquihire exploration

## Common Pivot Pitfalls

- [ ] Keeping the old product alive "just in case" (drains focus)
- [ ] Not letting go of team members who were critical to old but not new
- [ ] Over-communicating to existing customers (not every change needs a call)
- [ ] Investors who supported old product and are now skeptical — address directly
- [ ] Pivoting the messaging before the product (customers see through)
