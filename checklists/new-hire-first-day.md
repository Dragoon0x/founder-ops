# New Hire First Day

Small details set tone. Someone who had a great Day 1 remembers it for years.

## Night Before

- [ ] Welcome message in Slack pre-scheduled for 9 AM
- [ ] Workspace prepped (if in-person: desk, monitor, clean)
- [ ] Laptop arrived + confirmed working
- [ ] Software access queued (will activate Day 1 morning)
- [ ] 1:1 with manager scheduled for end of Day 1

## Day 1 Morning (by 9 AM)

### Access (should JUST WORK)
- [ ] Email (they can send/receive)
- [ ] Slack (joined, introduced)
- [ ] Calendar (shared with team)
- [ ] Core product access (can actually use what we make)
- [ ] Code repo access (if eng)
- [ ] Design tools (if design)
- [ ] Password manager (1Password, Bitwarden)
- [ ] HR portal (benefits, payroll)

### Welcome
- [ ] Manager greeting (in-person or video call)
- [ ] Slack intro in #general (manager posts, keeps it genuine)
- [ ] Intro meetings scheduled for week
- [ ] Lunch planned (team, or lunch with manager 1:1)

### Documents (20-30 min read)
- [ ] Mission / vision doc (1 page)
- [ ] Company handbook / operating principles
- [ ] Role scorecard (what they'll be measured on)
- [ ] 30-60-90 plan draft (editable by them)
- [ ] Who's who guide (team + rough org chart)

## Day 1 Afternoon

### Set up
- [ ] Dev env setup if eng (should be 2-hour setup max — if longer, we've failed)
- [ ] Tools walkthrough if needed
- [ ] First small task identified (shipable within week)

### People
- [ ] Intros with 2-3 key people they'll work with
- [ ] Manager 1:1 (30 min, not about tasks — about vibe)

## End of Day 1

### Manager 1:1 (30 min) — Questions to ask

- [ ] Was today's onboarding good? What could've been better?
- [ ] Anything unclear or worrying?
- [ ] What's one thing you want me to know about how you work best?
- [ ] What would make this week feel like a win for you?
- [ ] Any access still broken?

### Manager's own check
- [ ] They looked energized by end of day (not drained from navigating broken systems)
- [ ] They got at least one meaningful interaction with a non-manager
- [ ] They're clear on what they'll do tomorrow

## If Remote (additions)

- [ ] Virtual welcome tour of tools / channels
- [ ] Slack etiquette guide (we respond in X hours, async default)
- [ ] Remote work stipend info (if offered)
- [ ] Home-office setup confirmed working (camera, mic, ergonomics)

## Common Day-1 Failures (avoid)

- Laptop not ready
- Tools access broken
- No one waiting / greeting
- Overscheduled (10+ intro meetings back-to-back = exhausting)
- Underscheduled (they sit alone wondering what to do)
- Stack of docs to read with no context
- "Dive in and start coding" with no onboarding
- No 1:1 with manager
