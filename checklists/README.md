# Checklists

Structured checklists for high-stakes, infrequent, high-cost-of-error
operations. These aren't wall-of-text guidance. They are printable,
go-down-the-list artifacts.

Use them when:
- The operation is infrequent (so you won't remember the pattern)
- The cost of missing a step is high
- Multiple people are involved (so shared source of truth matters)

Don't over-use them:
- Daily/weekly workflows → run them as habit, not as checklist
- Checklists for checklists → you've over-engineered

## Format

Each checklist uses:
- [ ] Checkable items
- Grouping by phase / timeline
- Ownership indicator where multi-person
- "Why this matters" when non-obvious
