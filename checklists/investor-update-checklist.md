# Investor Update Checklist

Every month. Short. Specific. Useful.

## Before Writing

- [ ] Last update reviewed (what did you promise?)
- [ ] This month's raw data pulled (metrics, wins, misses)
- [ ] Ask list for investors compiled
- [ ] Decision: who gets the update (all investors, or tiered?)

## Structure Check (use `prompts/investor-update-monthly.md`)

- [ ] Subject line: "[Company] — [Month Year] Update"
- [ ] TL;DR at top (3 bullets max)
- [ ] Metrics table (same metrics every month — shows trend)
- [ ] Highlights (3-5)
- [ ] Lowlights (be honest — 2-3)
- [ ] Team update
- [ ] Specific asks
- [ ] Forward look (next month priorities)

## Content Gut-Check

- [ ] Am I being honest about what didn't work?
- [ ] Are the numbers the same metrics I reported last month?
- [ ] Am I asking for specific, actionable help?
- [ ] Would an investor read this and think "I want to be more involved"?
- [ ] Length: 400-800 words (not 2000)

## Length Check

- [ ] Under 800 words total
- [ ] Metrics table is <10 rows
- [ ] Highlights/lowlights are 3-5 items each
- [ ] Asks are 2-3 specific things

## Before Sending

- [ ] Spell-check
- [ ] Numbers double-checked against your dashboard
- [ ] "Last updated" date correct
- [ ] Graph images attached (if any)
- [ ] Subject line final

## Distribution

- [ ] BCC investors (never CC — protects their emails)
- [ ] Send from founder's address (not a generic one)
- [ ] Timing: first week of each month (consistency matters)
- [ ] Follow-up cadence: don't chase responses; they're reading even if not replying

## After Sending

- [ ] Track who opens (if you can — don't over-analyze)
- [ ] Respond to replies within 48h
- [ ] Note asks that got responses (actionable intros, feedback)
- [ ] Save to `output/investor-updates/` for next-month reference

## Monthly Rhythm

- Week 1 of month: Write + send last month's update
- Week 2-4: Execute
- End of month: Collect data, draft for next update

## Pitfalls

- [ ] Don't hide bad months — investors know from metrics anyway
- [ ] Don't make asks only when you need help (quiet month asks build relationships)
- [ ] Don't change metric definitions mid-stream (shows you're hiding)
- [ ] Don't write novels (time they spend should be < 5 min read)
- [ ] Don't CC mass-blast — BCC is the professional norm

## Quarterly Upgrade

Once a quarter, the update becomes longer (800-1500 words):
- Quarterly retrospective
- Goal → actual for quarterly plan
- Strategic update (not just tactical)
- Longer-form asks
