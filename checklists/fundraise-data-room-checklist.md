# Fundraise Data Room Checklist

What every serious investor will want to see during diligence.

## The Minimum (pre-data-room ask: share pitch deck + 1-pager)

- [ ] Pitch deck (clean, no typos)
- [ ] One-pager summary
- [ ] Founder backgrounds (linkedin or bio)

## The Data Room (shared when investor signals serious interest)

Use Docsend, Notion, or Google Drive with view/download permissions you control.

### 1. Company Basics
- [ ] Incorporation docs (state of incorp, date)
- [ ] Cap table (clean — see `library/cap-table-mistakes.md`)
- [ ] Prior investment documents (SAFEs, notes, priced rounds)
- [ ] Board composition + bylaws
- [ ] Summary of outstanding options + pool
- [ ] 409A valuation (if recent)

### 2. Financials
- [ ] Historical P&L (12+ months of monthly data)
- [ ] Current month metrics dashboard
- [ ] Projected financial model (18-24 months)
  - Assumptions explicitly stated
  - Revenue build-up (customer by customer if small)
  - Cost structure by category
- [ ] Burn rate + runway history
- [ ] Bank statements (redacted if needed) or accounting software access

### 3. Revenue & Customer
- [ ] Customer list with ACV + start date
- [ ] Revenue concentration: top 5/10 customers as % of ARR
- [ ] Churn log (when, who, why, revenue impact)
- [ ] Cohort retention analysis
- [ ] Win/loss analysis (last 20 deals)
- [ ] 3-5 customer case studies or quotes (written permission)

### 4. Product
- [ ] Roadmap (next 12 months)
- [ ] Product demo video or live access
- [ ] Key product metrics (MAU, DAU, retention, feature adoption)
- [ ] Tech stack / architecture (one-pager is enough)
- [ ] AI usage / data pipeline (relevant for 2026)

### 5. Go-to-Market
- [ ] GTM motion description (sales-led, PLG, hybrid)
- [ ] Channels + CAC by channel
- [ ] Sales cycle length + conversion rates
- [ ] Sales team structure
- [ ] Pipeline snapshot (current deals, stages)
- [ ] Pricing + packaging + discount policy

### 6. Team
- [ ] Org chart
- [ ] Key hires made in last 12 months (with comp, equity)
- [ ] Open roles with JDs
- [ ] Founder + exec equity
- [ ] Employee handbook / cultural docs
- [ ] Compensation philosophy

### 7. Market
- [ ] Competitive landscape (honest, not cherrypicked)
- [ ] Positioning vs. alternatives
- [ ] TAM/SAM/SOM analysis (bottom-up preferred)
- [ ] Market timing thesis

### 8. Legal
- [ ] Contract templates (customer, employee, advisor)
- [ ] IP assignment confirmation (all code/design assigned to company)
- [ ] Trademark / patent status
- [ ] Data / privacy compliance (GDPR, CCPA)
- [ ] Customer contracts with key terms flagged (MFN, exclusivity, etc.)
- [ ] Any litigation / claims disclosure

### 9. References
- [ ] 3-5 customer references (pre-warned, willing to take a call)
- [ ] 3-5 investor references (from prior rounds)
- [ ] 2-3 partner references (advisors, board members)

### 10. Founder / Team Backchannels
- [ ] List of 5+ people who'd take a reference call on CEO
- [ ] List for each exec

## Red Flags in Your Own Data Room

If any of these are true, FIX before sharing:
- [ ] Cap table inconsistencies (founder math wrong)
- [ ] Financials from different source-of-truth numbers
- [ ] Missing churn data (looks like you're hiding)
- [ ] Revenue concentration >30% in one customer (discuss proactively)
- [ ] No documented IP assignment from all contributors

## How Investors Use the Data Room

- Junior analyst reviews first (2-4 hours)
- Partner spot-checks 3-4 areas
- Questions come back — respond within 24h
- Customer calls happen
- Final partnership review = your data room is referenced

## Access Controls

- [ ] View-only (no download) by default
- [ ] Watermark every file with investor name
- [ ] Expiring links (1-2 weeks access)
- [ ] Track views (who looked at what, how long)
- [ ] Staged access (level 1 first, deeper on second meeting)
