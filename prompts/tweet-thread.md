# Prompt: Tweet Thread

## Output
`output/tweet-[topic].md`

---

## Prompt Template

```
I want to write a tweet thread about [TOPIC].

The core insight I want to share:
[ONE SENTENCE — the ONE thing someone should walk away with]

My audience:
[WHO reads my stuff — be specific]

Supporting material:
[DATA / STORIES / EXAMPLES I can use]

Desired length:
[SHORT (5-7 tweets) / MEDIUM (8-12 tweets) / LONG (13-20 tweets)]

Run /founder-ops content tweet-thread and produce:

## THREAD

Tweet 1 (Hook):
- Under 240 chars
- Specific outcome or number
- Teases the lesson without giving it away
- No "Let me tell you about..." / no "Here's a thread on..."
- 3 variants of tweet 1

Tweets 2-N (Body):
- Each tweet standalone readable
- Line breaks inside tweets for readability (don't wall-of-text)
- Concrete examples > abstract claims
- 1 tweet = 1 idea

Final tweet (CTA):
- Specific action (reply, share, DM)
- NOT "follow me for more" (desperate)
- Link if relevant

## PATTERNS TO USE (pick 1-2)
- Numbered list ("5 things I learned...")
- Contrarian take ("Everyone says X. They're wrong because Y")
- Story ("3 years ago I did X. Here's what I learned")
- Breakdown ("Here's how we got from X to Y")
- Comparison ("Most founders do X. The best do Y")

## PATTERNS TO AVOID
- "Let me break it down for you"
- "Here's the thing"
- All-uppercase CTA
- Emojis as structure (👇🏽 or 🧵)
- Adding "1/ 2/ 3/" numbering (use line breaks instead, it's native)
- Ending every tweet with "..." (looks incomplete)

## ENGAGEMENT PATTERNS
- Reply hook in the middle — a question people will answer
- Sharable fact or quote in the middle — bookmark bait
- Specific call to action at end

## VERSION VARIANTS
- A: Data-heavy (more numbers, less story)
- B: Story-heavy (more narrative, fewer numbers)
- Pick one to publish; other as backup

Save as output/tweet-[TOPIC].md.
```
