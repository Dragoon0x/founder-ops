# Vendor Procurement Workflow

## Trigger
Need for a new tool, service, or external provider.

## Phase 1: Requirements (Day 1-2)

### Define before you shop
- **Problem statement**: What specific workflow is broken or manual?
- **Must-haves**: 3-5 non-negotiable capabilities
- **Nice-to-haves**: Features that would be great but aren't blockers
- **Budget ceiling**: Max monthly/annual spend approved
- **Users**: Who will use this? How many seats?
- **Integration needs**: What existing tools must it connect with?
- **Data sensitivity**: Will customer data flow through this vendor?

### Build vs buy decision
Before evaluating vendors, ask:
- Could we build this in < 2 weeks of eng time?
- Is this core to our value prop? (If yes, probably build)
- Will we need deep customization? (If yes, probably build)
- Is this a solved problem with mature vendors? (If yes, probably buy)

## Phase 2: Evaluation (Day 3-7)

### Shortlist construction
- Research 5-8 options (G2, Product Hunt, peer recommendations)
- Eliminate based on must-haves (usually cuts to 3-4)
- Request demos from top 3

### Demo evaluation scorecard
Rate 1-5 on each:
- Feature completeness for your use case
- UX quality (will your team actually use this?)
- Integration capability (API quality, native integrations)
- Pricing transparency (hidden costs are a red flag)
- Support quality (response time during the sales process is a signal)
- Security posture (SOC 2, data residency, encryption)

### Trial period
- Always do a paid trial or free trial before committing
- Minimum 2 weeks, ideally 30 days
- Have 2-3 actual users test real workflows, not hypothetical scenarios
- Document friction points during trial

## Phase 3: Negotiation (Day 8-10)

### Negotiation levers
- **Annual vs monthly**: Annual commitment typically gets 15-25% discount
- **Multi-year**: 2-year deal can get 25-40% discount (but risky for startups)
- **Seat count**: Commit to a growth plan for better per-seat pricing
- **Payment terms**: Net 30/60 is standard. Ask for quarterly billing if cash flow is tight
- **Out clause**: Negotiate a 30-day termination clause in the first year

### Red flags in vendor contracts
- Auto-renewal without 60-day notice window
- Price increase caps absent (they can raise prices 50% at renewal)
- Data portability not guaranteed (your data is hostage)
- SLA without financial penalties (meaningless)
- Non-compete or exclusivity clauses

## Phase 4: Onboarding (Day 11-14)

### Rollout checklist
- [ ] Admin access configured with SSO
- [ ] Integrations connected and tested
- [ ] Data migration completed (if switching from another vendor)
- [ ] Team training scheduled (30-60 min session)
- [ ] Documentation written (internal how-to, not vendor docs)
- [ ] Success metrics defined (adoption rate, time saved, error reduction)
- [ ] 30-day review scheduled

### Ownership
Assign a vendor owner: one person responsible for the relationship, renewals, and performance monitoring. Without this, vendor costs bloat silently.
