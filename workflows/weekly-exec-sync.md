# Weekly Exec Sync Workflow

## Trigger
Every Monday, 10:00 AM. Non-negotiable unless a co-founder is traveling (reschedule within 24 hrs).

## Pre-conditions
- Each function lead posts their weekly snapshot by Sunday 6 PM:
  - 3 key metrics (actual vs target)
  - Top win
  - Top risk
  - One ask of another function

## Agenda Structure (45 min)

### 1. Scorecard review (10 min)
Pull up the shared metrics dashboard. Review only RED and YELLOW metrics.
Green metrics get zero airtime. If everything is green for 3 consecutive weeks, your targets are too soft.

**RED metric protocol:**
- Owner states: what happened, why, what's the recovery plan, when will it be green
- If no recovery plan exists, it becomes the #1 priority for that function this week

### 2. Cross-functional asks (15 min)
Each function lead's "one ask" gets addressed:
- Immediate yes/no if possible
- If needs investigation, assign owner + deadline (max 48 hrs)
- If it's a resource conflict, escalate to prioritization (step 3)

### 3. Prioritization conflicts (10 min)
Any week where two functions need the same resource (usually engineering):
- Each side states impact in revenue or retention terms
- Decision made by CEO, documented in decision log
- Losing side gets a specific "when" for their request

### 4. Founder-only check-in (10 min)
Non-exec attendees drop. Co-founders only.
- "What's the thing we're not talking about?"
- "What decision are we avoiding?"
- "How are you actually doing?"

## Exit Criteria
- All RED metrics have recovery plans with owners
- All cross-functional asks resolved or assigned
- Decision log updated
- No open prioritization conflicts

## Cadence Health Signals
| Signal | Healthy | Warning |
|--------|---------|---------|
| Meeting runs over | < 1x/month | > 2x/month |
| Metrics not submitted on time | < 1 person | > 2 people |
| Same RED metric 3+ weeks | Never | Happening now |
| Founder check-in skipped | Never | 2+ consecutive |
