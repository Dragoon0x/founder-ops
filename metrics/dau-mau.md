# DAU/MAU Ratio (Stickiness)

## Definition
The percentage of monthly active users who use the product on any given day.

## Formula
```
DAU/MAU = Daily Active Users / Monthly Active Users × 100
```

## First: define "active"
"Active" is NOT just "logged in" or "opened the app."
Active = performed a core action that delivers value.

| Product Type | Core Action |
|-------------|-------------|
| Messaging app | Sent a message |
| Analytics tool | Viewed a dashboard or ran a query |
| Project management | Updated a task or commented |
| E-commerce | Browsed products or made a purchase |
| Social media | Posted, commented, or reacted |
| Design tool | Edited a file |

## Benchmarks
| DAU/MAU | Rating | Examples |
|---------|--------|---------|
| > 50% | Exceptional | Messaging apps, daily work tools (Slack: ~70%) |
| 30-50% | Strong | Social media, productivity tools |
| 15-30% | Average | Most B2B SaaS, content platforms |
| < 15% | Low | Monthly-use tools, reporting, compliance |

Low DAU/MAU isn't inherently bad. A payroll tool used once a month is fine. The metric matters most for products that should be daily-use.

## Why it matters
High DAU/MAU = habit. Habits predict retention.
Products with DAU/MAU > 40% typically have annual retention > 80%.

## Improvement levers
1. **Notifications**: Bring users back with relevant, timely triggers (not spam)
2. **Daily use cases**: Add features that require or reward daily engagement
3. **Collaboration**: Multi-user features increase daily touchpoints
4. **Content refresh**: New content daily (feeds, recommendations, updates)
5. **Streaks/gamification**: Use sparingly; effective for consumer, less for B2B

## Alert thresholds
- DAU/MAU declining for 3+ months (product is becoming less sticky)
- DAU/MAU < 10% for a product expected to be daily-use
- DAU flat while MAU grows (new users aren't forming habits)
