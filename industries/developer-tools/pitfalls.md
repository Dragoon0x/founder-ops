# Developer Tools — Common Founder Pitfalls

## Product / DX Pitfalls

| Pitfall | Why it kills you | Fix |
|---------|------------------|-----|
| Slow first run | Devs leave in 5 minutes | Setup must be <5 min, ideally <1 min |
| Bad docs | Devs think product is bad | World-class docs from day 1 |
| Hidden pricing | Devs assume "expensive" | Pricing on homepage |
| Sign up to try | Devs leave immediately | Try before signup if possible |
| No CLI for complex tools | Devs prefer terminal | Build CLI alongside web UI |
| No SDK / API | Limited integration | API-first, then UI |
| Poor error messages | Frustration | Clear errors with actionable fixes |
| No status page | Trust problem | Public status page, real-time |

## GTM Pitfalls

| Pitfall | Why it kills you | Fix |
|---------|------------------|-----|
| Marketing-speak landing page | Devs check out | Technical, specific, code samples |
| No "show, don't tell" | Devs need to see the code | Embed real code on landing page |
| Generic content | Doesn't reach devs | Deeply technical content (devs share) |
| Sales-first motion | Kills bottoms-up | Self-serve must work |
| No free tier (or one too restrictive) | Can't try meaningfully | Free tier must enable real evaluation |
| Aggressive upsell | Devs leave for FOSS alternative | Clear paid value, not artificial limits |
| Forms gating docs / examples | Devs leave | All docs and examples public |

## Community Pitfalls

| Pitfall | Why it kills you | Fix |
|---------|------------------|-----|
| Building community for vanity | Empty Discord, dead Slack | Wait until you have product to discuss |
| Ignoring GitHub issues | Devs feel unheard | Respond to every issue, even with "we'll consider" |
| No community guidelines | Toxicity emerges | Clear guidelines + active moderation |
| Buying a community | They leave when you stop paying | Earn it slowly through value |
| Founder absent from community | Loses authenticity | Founder visible in Discord/Slack/issues |

## OSS-Specific Pitfalls (if open source)

| Pitfall | Why it kills you | Fix |
|---------|------------------|-----|
| Unclear monetization plan | Either commercial floundering or OSS goodwill loss | Decide early: open core, COSS, services |
| License confusion | Loses contributors and customers | Clear license; communicate freely vs restricted features |
| Treating contributors as employees | Burnout | Recognition, not exploitation |
| Pivoting to closed source | Massive trust loss (see HashiCorp, Elastic backlash) | Don't if you can avoid it |
| Building both OSS and SaaS without strategy | Dilutes both | Clear differentiation between them |

## Pricing Pitfalls

| Pitfall | Why it kills you | Fix |
|---------|------------------|-----|
| Per-seat for individual-use tools | Doesn't fit usage pattern | Usage-based or organization-based |
| Per-API-call when usage is bursty | Customers can't predict bills | Tiered usage with predictable cap |
| Enterprise gating critical features (SSO, audit logs) | Common but generates resentment | Justify with comms; offer mid-tier |
| Free tier so generous nobody upgrades | No revenue | Free tier ends at meaningful threshold |
| No usage-based even when usage scales | Limits expansion revenue | Hybrid pricing |
| Surprise overage fees | Bills that destroy customer trust | Email warnings, hard caps option |

## Sales / Enterprise Pitfalls

| Pitfall | Why it kills you | Fix |
|---------|------------------|-----|
| AEs without dev background | Can't sell technically | Hire technical AEs or sales engineers |
| Demos that ignore the developer | Selling to executives ignoring users | Multi-stakeholder demos |
| No solutions engineering function | Enterprise deals stall in technical eval | First SE around $2-5M ARR |
| Long custom contracts | Slow procurement | Standard MSA + click-through SLA |
| No SOC 2 by Series A | Enterprise deals die | Start SOC 2 process at $2M ARR |
| No SAML/SSO | Can't sell to enterprises | Build before enterprise pursuit |

## Strategic Pitfalls

| Pitfall | Why it kills you | Fix |
|---------|------------------|-----|
| Targeting same buyer as everyone | Crowded | Find an underserved segment |
| Building "just another X" | No differentiation | DX or specific use case advantage |
| Ignoring Cursor / Copilot trend | AI-augmented dev is changing everything | Adapt or get displaced |
| Skipping fundamentals for hype | Vector DBs without actual customers, etc | Solve real problems, not narrative ones |
| Hiring senior PMs early | Devs distrust process | Engineer-led product until $5M ARR |
