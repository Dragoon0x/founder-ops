# Reports

Real example reports showing what good founder-generated reports look like. Use these as reference when generating your own through `founder-ops` modes.

Every report here is anonymized and composite — numbers are illustrative, not from a single real company.

## Principles across all reports

1. Numbers first, narrative second. If you're burying the metric, you're hiding from it.
2. Name the bad news in the first third. Investors and teams trust you more, not less, for doing this.
3. One decision or ask per section. Don't stack three asks into one paragraph.
4. What changed since last time. Reports build on each other. Show the delta.
5. What we learned, not what we did. A list of activities is a timesheet, not a report.
