# Decision Trees

Explicit decision logic for the recurring founder calls. Each file walks the decision point by point with concrete criteria, not vague guidance.

Structure for each:
1. **Context** — when this decision arises
2. **The tree** — branching logic with specific thresholds
3. **Tiebreakers** — what to check when stuck at a node
4. **Common wrong answers** — what people pick under pressure
5. **Reversibility** — can you undo, and at what cost

## How to use
When facing one of these decisions, sit down, open the file, walk the tree honestly. Don't skip a node because you already "know" the answer — if you know the answer, you're usually wrong about at least one branch.
