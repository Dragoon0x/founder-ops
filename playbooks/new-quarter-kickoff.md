# Playbook: New Quarter Kickoff

## When to Run
First week of every quarter. Allocate 4-6 hours over 2-3 days.

## The Sequence

### Day 1: Reflect (2 hours)
**Step 1:** Run `/founder-ops retro` (quarterly mode)
- Score the previous quarter against goals
- Identify what worked / what didn't / what surprised
- Document decisions made and their outcomes

**Step 2:** Run `/founder-ops metrics`
- Update the metrics dashboard
- Build month-over-month and quarter-over-quarter views
- Identify trend changes

### Day 2: Plan (2 hours)
**Step 3:** Run `/founder-ops strategy` (or `/founder-ops prioritize` if at smaller scale)
- Set 3-5 quarterly objectives
- Assign owners
- Define success criteria

**Step 4:** Run `/founder-ops roadmap`
- Map the priorities to product/eng work
- Sequence based on dependencies
- Identify what we're explicitly NOT doing

### Day 3: Communicate (1-2 hours)
**Step 5:** Run `/founder-ops update`
- Draft monthly investor update with quarterly framing
- Include: previous quarter results, new quarter goals

**Step 6:** Run `/founder-ops internal-comms`
- Plan the quarterly all-hands
- Draft the message for the team
- Schedule the meeting

## Output Artifacts
- `reports/retro-quarter-[Q]-[year].md`
- `data/metrics-quarter-[Q].md`
- `output/strategy-quarter-[Q].md`
- `output/roadmap-quarter-[Q].md`
- `output/investor-update-[month].md`
- `output/all-hands-script-quarter-kickoff.md`

## Done Right Looks Like
- Whole team knows top 3 priorities by EOW1
- Investors received a clear update by EOW1
- Metrics dashboard reflects new quarter targets
- Roadmap committed to and shared
