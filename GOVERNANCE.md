# Governance

## Project Status

founder-ops is an **open-source community project** maintained by Dragoon0x
and contributors. It is NOT a commercial product. There's no company behind
it. It's a free gift to the founder community.

## Decision-Making

### Decisions by Maintainer (default)
- Day-to-day PRs
- New modes that fit existing architecture
- Bug fixes
- Documentation updates
- Dependency updates

### Decisions by Community Vote
- Breaking changes to the contract
- New top-level categories (new playbook types, new framework categories)
- Governance changes

### Votes
- Open a GitHub Discussion
- 2-week open comment period
- Maintainer makes final call if no clear consensus

## Contributions

See `CONTRIBUTING.md`. Short version:
- PRs welcome, but high bar
- Follow the structure of existing files
- New modes: must include methodology + templates + anti-patterns
- No commercial promotion
- MIT-licensed only

## Code of Conduct

See `CODE_OF_CONDUCT.md`. Short version:
- Be kind
- Focus on ideas, not people
- No harassment
- English primary, but contributions in other languages welcome in discussions

## Roadmap

Not a formal product roadmap. Priorities driven by:
1. Gaps real founders report
2. Patterns from real startup failures
3. Frameworks proven in the wild
4. Translation of docs for non-English founders

## Forking

Forking is encouraged. If you build something interesting on top, we'll
link to it. Naming: please don't use "founder-ops" in the name to avoid
confusion.

## Archive / End-of-Life Policy

If the project becomes inactive (6+ months with no maintenance):
- GitHub Archive mode enabled
- Announcement with last-known-good version
- Links to active forks if any
- Repo preserved for historical use
