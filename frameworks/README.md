# Frameworks

Reusable scoring and decision-making frameworks. Each is a pure tool — load
the relevant one when running a mode that needs structured scoring.

## Available Frameworks

| Framework | Purpose | Used By |
|-----------|---------|---------|
| `RICE.md` | Score features by Reach × Impact × Confidence / Effort | prioritize, product, roadmap |
| `ICE.md` | Score quickly by Impact × Confidence × Ease | prioritize, growth |
| `MoSCoW.md` | Bucket items by Must/Should/Could/Won't | prioritize, product |
| `KANO.md` | Categorize features by Basic/Performance/Delight | product, positioning |
| `JTBD.md` | Jobs-to-be-Done framework | discovery, positioning, product |
| `SWOT.md` | Strengths/Weaknesses/Opportunities/Threats | strategy, compete |
| `OKR.md` | Objectives + Key Results | strategy, retro, roadmap |
| `KPI-tree.md` | Tree of metrics decomposition | metrics, finance |
| `cynefin.md` | Decision-making in complex domains | strategy, crisis |
| `eisenhower.md` | Urgent/Important matrix | calendar, prioritize |
| `north-star-metric.md` | How to define and use a North Star | metrics, strategy |
| `rfm-segmentation.md` | Recency-Frequency-Monetary customer segmentation | customer-success, retention |
| `ansoff-matrix.md` | Existing/New markets × Existing/New products | strategy, growth |
| `bcg-matrix.md` | Stars/Cash Cows/Question Marks/Dogs | strategy, prioritize |
| `5-whys.md` | Root cause analysis | retro, debugging, post-mortem |
