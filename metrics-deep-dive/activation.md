# Activation

The first moment a user realizes the product's value. Get this wrong, retention dies.

## Definition

Activation = user reaches the "aha moment" that makes them likely to return.

Examples:
- Slack: sends 2,000 messages in a team (Slack's original benchmark)
- Facebook: 7 friends in 10 days
- Dropbox: uploads a file
- Zapier: creates and runs first Zap
- Figma: first edit on a shared file
- Notion: second return in first week

Every product has an activation action. Finding it is the first ops problem of any PLG product.

## How to find yours

1. Pull 100 users who retained to 30 days.
2. Pull 100 users who churned in week 1.
3. Compare behaviors in first 24-72 hours.
4. Find behaviors that distinguish at >2:1 ratio (retained users did X at 60%, churned did X at 20%).
5. That behavior is your activation candidate.

## Activation rate

Activation rate = % of new users who complete the activation action.

Benchmark: 30-50% for PLG. Below 25% = activation is too hard or the wrong action.

## Time to activate

Time to activate (TTA) = time from signup to activation.

Benchmark: <24 hours for PLG, <7 days for SaaS, <15 minutes for consumer.

TTA > 30 days means your activation action is too high-friction. Users forget.

## Single-metric vs multi-metric activation

**Single-metric:** one clear action ("sent first message")

**Multi-metric:** composite of actions ("invited teammate" + "connected data source" + "used feature X")

Single is simpler. Multi is more predictive. Usually end up with 2-3 actions that together define activation.

## Activation drivers

1. **Time-to-value of the action itself.** If the first valuable thing takes 15 steps, fix that.
2. **Onboarding design.** Guided onboarding beats empty state every time.
3. **Product personalization.** Use signup info to prefill, suggest, accelerate.
4. **Social proof in product.** "Other teams started by…"
5. **Checklist / progress bar.** Creates commitment.
6. **Trigger moments.** Sample data, templates, demo environment.

## Common mistakes

- Defining activation too low ("signed up" — everyone does this, means nothing)
- Defining activation too high ("used product daily for 30 days" — that's retention, not activation)
- Not segmenting by persona (admins activate differently from end users)
- Not measuring by channel (paid users activate differently from organic)

## Activation and retention

Activated users retain 3-10x better than non-activated users. Every $1 invested in activation has outsized return because it prevents churn before it starts.

If activation rate is 30%, pulling it to 50% more than 50% increases active user base (compounds through retention).

## Measurement cadence

Weekly: activation rate by cohort, by channel, by persona.
Monthly: time-to-activate distribution, churn rate of activated vs non-activated.
Quarterly: re-evaluate definition — did the product change? Did ICP shift?
