# Financial Models

Opinionated, stage-aware financial models for founders. Every model here assumes you know the inputs — if you don't, go run the underlying diagnostic first.

## What's here

- `saas-three-statement.md` — P&L, cash flow, balance sheet tied together
- `burn-model.md` — how to model burn with scenarios
- `cac-ltv-model.md` — unit economics that don't lie
- `cohort-revenue-model.md` — bottoms-up forecasting
- `cap-table-scenarios.md` — dilution math across rounds
- `409a-valuation.md` — what drives strike prices
- `term-sheet-economics.md` — liquidation preference math, real
- `exit-waterfall.md` — who gets paid what at exit
- `fundraise-sizing.md` — how much to raise, really
- `hiring-plan-model.md` — cost of every hire by function
- `runway-extension-moves.md` — levers and their real impact
- `unit-economics-by-model.md` — SaaS, marketplace, consumer, devtools
- `marketplace-gmv-model.md` — take rate, liquidity, supply/demand
- `consumer-subscription-model.md` — LTV with churn curves
- `hardware-bom-model.md` — COGS, margin, working capital

## Principles

1. Model the downside first. Base case lies.
2. Sensitivity tables over point estimates.
3. Tie revenue forecasts to hiring triggers, not wishes.
4. Every assumption gets a source or a date-stamped guess.
5. Re-forecast monthly. The model is a living thing.
