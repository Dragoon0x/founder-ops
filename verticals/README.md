# Verticals — Industry-Specific Operating Guides

Each vertical has dynamics that override general startup advice. These
guides layer on top of the core 49 modes with industry-specific
overrides for the modes that matter most.

| Vertical | Best for |
|----------|----------|
| `saas-b2b.md` | Selling software to other businesses |
| `marketplace.md` | Two-sided marketplaces (supply meets demand) |
| `consumer.md` | B2C apps, DTC, subscription consumer |
| `fintech.md` | Financial services (lending, payments, neobanks) |
| `healthtech.md` | Healthcare, digital health, providers/payers/patients |
| `deeptech.md` | Hardware, robotics, biotech, climate, deep R&D |
| `devtools.md` | Tools developers buy |
| `ai-products.md` | AI-native software (LLM-powered) |

Each guide covers:
- Stage-by-stage operating priorities
- Mode overrides (where the vertical changes the playbook)
- Vertical-specific anti-patterns
- Reference benchmarks (linked to `benchmarks/`)
- "If you're new to this vertical, read this first" intros
