# Diagnostic: Product Health

Run quarterly. Takes 3 hours with analytics access.

## Scorecard

### User Engagement (25 points)
- DAU/MAU ratio: 5 pts if > 40%, 1 pt if < 15%
- Session duration trend: 5 pts if stable/increasing, 1 pt if declining > 20%
- Feature breadth: 5 pts if users engage with 3+ features weekly, 1 pt if single-feature usage
- Power user % (users using advanced features): 5 pts if > 20%, 1 pt if < 5%
- Time to value: 5 pts if < 5 min to core action, 1 pt if > 30 min

### Retention (25 points)
- D1 retention: 5 pts if > 60%, 1 pt if < 30%
- D7 retention: 5 pts if > 40%, 1 pt if < 20%
- D30 retention: 5 pts if > 25%, 1 pt if < 10%
- Retention curve shape: 5 pts if flattens (PMF signal), 1 pt if continuous decline
- Cohort improvement: 5 pts if newer cohorts retain better, 1 pt if worse

### Quality (25 points)
- Bug report volume: 5 pts if declining MoM, 1 pt if increasing
- Crash rate (mobile/desktop): 5 pts if < 0.1%, 1 pt if > 1%
- Page load time (p95): 5 pts if < 2s, 1 pt if > 5s
- Error rate (API): 5 pts if < 0.1%, 1 pt if > 1%
- Uptime: 5 pts if > 99.95%, 1 pt if < 99.5%

### Growth Signals (25 points)
- Organic signup rate: 5 pts if > 30% of new users organic, 1 pt if < 10%
- Viral coefficient: 5 pts if > 0.5, 1 pt if < 0.1
- Feature request volume: 5 pts if healthy demand with clear patterns, 1 pt if sparse or scattered
- NPS: 5 pts if > 50, 1 pt if < 20
- Competitive win rate (product-attributed): 5 pts if > 60%, 1 pt if < 30%

## Scoring
- 80-100: Strong product. Focus on growth and expansion.
- 60-79: Solid foundation. Address retention or quality gaps.
- 40-59: Needs investment. Major product work needed before scaling.
- < 40: Critical. Reassess PMF and product strategy.
